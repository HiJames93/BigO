# 几种常见时间复杂度分析：
计算时间复杂度 往往是计算比较大的  而且是不确定的数，如果已经确定了，那么就不用计算了，也是我们说的常量。

-  常数：O(1) 表示是常数，所有能确定的数字我们都用O(1)， O(1000)=>O(1)
- 对数：O(logn)，O(nlogn)
- 线性：O(n)
- 线性对数：O(nlogn)
- 平方：O(n^2)
- N次方：O(n^n)

我们怎么找时间复杂度：
1. 找有循环的地方
2. 找有网络请求（RPC、）的地方。
就是测试时间： log打印，计算平均时间。

# 几种常见的时间复杂度总结

O(1)>O(logn)>O(n)>O(nlogn)>O(n^2)>O(n^x)

登录： 判断用户名密码O(1)
列表的接口：排序，冒泡排序：=>找优秀的排序算法 快速排序，归并排序，堆排序
优化的目标就是要往O(1)的方向接近。
